Basic Architecture

HAProxy typically sits between clients and backend servers:


Client ---> HAProxy ---> Web Server 1
                     ---> Web Server 2
                     ---> Web Server 3

HAProxy intelligently routes requests based on load balancing algorithms like roundrobin, leastconn, or source.

Example Configuration

A minimal HAProxy configuration might look like this:


global
    log /dev/log local0
    maxconn 2048

defaults
    mode http
    timeout connect 5000ms
    timeout client  50000ms
    timeout server  50000ms

frontend http_front
    bind *:80
    default_backend servers

backend servers
    balance roundrobin
    server web1 192.168.1.101:80 check
    server web2 192.168.1.102:80 check

Use Cases

  • Load balancing web servers or APIs
  • Providing failover and redundancy
  • SSL termination for secure web traffic
  • Reverse proxying and caching
  • Traffic routing between microservices
